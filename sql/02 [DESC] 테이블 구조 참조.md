## DESC 명령

- 테이블에 어떤 열이 정의되어 있는지 알 수 있음
- DESC는 SQL 명령이 아님

```bash
DESC sample21;
```

```bash
+----------+-------------+------+-----+---------+-------+
| Field    | Type        | Null | Key | Default | Extra |
+----------+-------------+------+-----+---------+-------+
| no       | int         | YES  |     | NULL    |       |
| name     | varchar(20) | YES  |     | NULL    |       |
| birthday | date        | YES  |     | NULL    |       |
| address  | varchar(40) | YES  |     | NULL    |       |
+----------+-------------+------+-----+---------+-------+
4 rows in set (0.01 sec)
```

- **Field** : 열 이름
- **Type** : 해당 열의 자료형
- **Null** : NULL 값을 허용할 것인지 아닌지 나타내는 제약사항 (Yes - NULL 허용)
- **Key** : 해당 열이 키로 지정되어 있는지
- **Default** : 기본값

## 자료형

### INTEGER

- 정수값을 저장할 수 있는 자료형
- 소수점 포함 불가

### CHAR

- 문자열을 저장할 수 있는 자료형
- 언제나 고정된 길이로 데이터가 저장됨 (’고정길이 문자열’)
- 열의 최대 길이를 지정해야 함
- 최대 길이보다 작은 문자열을 저장할 경우 공백문자로 나머지를 채운 뒤 저장

### VARCHAR

- 문자열을 저장할 수 있는 자료형
- 저장할 문자열의 길이에 맞춰 저장공간을 가변적으로 사용하여 저장 (’가변길이 문자열’)

### DATE

- 날짜 값 저장
- 연월일 데이터

### TIME

- 시간을 저장할 수 있는 자료형
- 시분초 데이터

이외에도 많은 자료형이  있다.
